# Security Policy

## 공개 전 필수 설정

- `SECRET_KEY`, `ADMIN_USERNAME`, `ADMIN_PASSWORD`는 환경변수로 설정하세요.
- 실제 계좌번호와 예금주는 `PAYMENT_ACCOUNT`, `PAYMENT_HOLDER`로 주입하세요.
- `data.json`, SQLite DB 파일, Excel 매출 내보내기 파일은 공개 저장소에 커밋하지 마세요.

## 알려진 한계

이 앱은 행사장 내부 운영을 전제로 만든 경량 도구입니다. 현재 구조는 정식 상용 POS나 장기 운영 서비스 수준의 보안을 목표로 하지 않습니다.

- 관리자 화면은 세션 로그인으로 보호됩니다.
- 일부 스태프용 API는 현장 운영 편의에 맞춰 설계되어 별도 사용자 권한 모델이 없습니다.
- 공개 인터넷에 배포할 경우 HTTPS, 강한 관리자 비밀번호, 비공개 URL이 아닌 실제 인증/권한 정책을 추가로 검토해야 합니다.

## 취약점 제보

개인 프로젝트 공개본이므로 별도 보안 제보 채널은 없습니다. 포크 또는 이슈에서 재현 조건과 영향 범위를 함께 남겨 주세요.
